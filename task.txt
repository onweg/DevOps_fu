тема 6. Разработка контейнера для обеспечения безопасности системы при
угрозе несанкционированного доступа при помощи сторонних сервисов (215)


1. Разработка технического задания по теме (1).
2. Разработайте приложение по варианту (2) на облачной платформе SaaS на примере
фреймворков Diango/Flask по шаблону.
3. Приложение должно включать 4 контейнера (капример): сервер піпт, сервер
обработки данных, сервер базы данных и сервер интерфейса адопиистрирования
базы данных, з также настройку для доступа к серверу піпх.
4. На сервере обработки данных реализовать алгоритм шителлектуальной обработки
данных, табличного и визуального представления по варианту (3), согласованному с преподавателем
5. Создать и развернуть минимум 4 контейнера используя Docker.
6. Настроить взаимодействие между контейнерами. Проверить работоспособност»
конфигураций. При необход мости провести настройку сети контейнеров.
7. Для реализации базовой конфигурации кластера, состоящего из одного пода с
четырьмя контейнерами, создать манифест config.yaml.
8. Развернуть кластер используя Kubernetes.
9. Сгенерировать данные, на которых будет тестироваться работа алгоритма
нителлектуальной обработки данных, согласовав их состав с преподавателем.
10. Проверить работоспособность приложения на стенерпрованном наборе данных.
